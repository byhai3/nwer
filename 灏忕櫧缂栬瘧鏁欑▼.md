# 短视频守护 - 小白编译教程

> 不用担心，跟着下面的步骤一步步来，**不需要会编程**，也能做出属于自己的 APK。

---

## 方式一：GitHub 自动构建（最简单，强烈推荐）

这个方法你只需要在网页上点几下鼠标，GitHub 的服务器就会自动帮你生成 APK。

### 第 1 步：下载项目代码

1. 先把 `ShortVideoGuard.zip` 下载到你的电脑上
2. 右键解压，得到一个 `ShortVideoGuard` 文件夹

### 第 2 步：注册 GitHub 账号（如果已有可跳过）

1. 打开 https://github.com
2. 点击右上角 **Sign up**，用邮箱注册（免费）
3. 按提示完成邮箱验证

### 第 3 步：创建代码仓库

1. 登录 GitHub 后，点击右上角 **+** 号，选择 **New repository**
2. 在 **Repository name** 框里输入：`ShortVideoGuard`
3. 最下面选择 **Private**（私人仓库）
4. 点击绿色按钮 **Create repository**

### 第 4 步：上传代码

1. 在新仓库页面，点击 **uploading an existing file** 链接
2. 把电脑上 `ShortVideoGuard` 文件夹里的所有文件/文件夹，**拖拽**到网页上传区域
   - 注意：不是拖整个 `ShortVideoGuard` 文件夹，而是打开它，把里面的所有内容拖进去
3. 页面滚动到底部，点击 **Commit changes**

### 第 5 步：触发自动构建

1. 上传完成后，点击顶部菜单 **Actions**
2. 你会看到左侧有个 **Build APK** 工作流，点击它
3. 在右侧点击 **Run workflow** 按钮（灰色），再点击绿色的 **Run workflow**
4. 等待大约 5-10 分钟（页面会自动刷新，看到绿色 ✅ 就是成功了）

### 第 6 步：下载 APK

1. 构建成功后，回到仓库主页
2. 点击顶部 **Actions**，进入刚才的运行记录
3. 页面最下方有个 **Artifacts** 区域，点击 **app-debug** 下载
4. 下载的 ZIP 解压后就是 `app-debug.apk`，传到手机上安装即可！

## 方式二：Android Studio 手动编译

如果你想自己本地编译，或者想修改代码后再编译，用这个方法。

### 第 1 步：下载 Android Studio

1. 打开 https://developer.android.com/studio
2. 点击 **Download Android Studio**
3. 下载完成后安装（Windows 双击安装包，Mac 拖到应用程序里）

### 第 2 步：打开项目

1. 启动 Android Studio
2. 在欢迎界面点击 **Open**（或者顶部菜单 File → Open）
3. 找到你解压好的 `ShortVideoGuard` 文件夹，选中它，点击 **OK**
4. Android Studio 会自动下载 Gradle 和其他依赖，**第一次会比较慢**（5-15 分钟，看网速），请耐心等待底部进度条完成

### 第 3 步：构建 APK

等底部不再转圈圈后：

1. 点击顶部菜单 **Build → Build Bundle(s) / APK(s) → Build APK(s)**
2. 等待右下角出现提示 **Build Analyzer detected...** 或者看到底部显示 **Build completed**
3. 点击提示中的 **locate** 链接，或者直接打开文件夹：
   ```
   ShortVideoGuard/app/build/outputs/apk/debug/
   ```
4. 里面的 `app-debug.apk` 就是你要的 APK 文件！

### 第 4 步：传到手机安装

1. 用微信、QQ、数据线、蓝牙等方式把 APK 传到手机
2. 在手机上点击 APK 文件
3. 如果提示"未知来源"，去设置里允许即可
4. 安装完成！

## 安装后的设置

APK 装好后，还需要开启权限才能正常工作：

1. **开启无障碍服务**：打开「短视频守护」App → 点"前往开启无障碍服务" → 在系统设置里找到「短视频守护」打开开关
2. **允许悬浮窗**（部分手机需要）：系统设置 → 应用管理 → 短视频守护 → 权限 → 显示悬浮窗 → 允许
3. 然后打开抖音/快手，当检测到不适宜内容时就会自动出现遮挡层

## 常见问题

**Q: GitHub Actions 构建失败了怎么办？**
A: 进入 Actions 页面，点击失败记录，查看错误日志。通常是配置文件格式问题，可以把错误信息截图发到技术论坛求助。

**Q: Android Studio 打开项目后一直在转圈？**
A: 第一次打开需要下载 Gradle（约 100MB+）和依赖库，耐心等待。如果超过 30 分钟还在转，检查网络是否稳定。

**Q: 安装 APK 时手机提示"解析包错误"？**
A: 可能是 APK 没传完整，重新传一次。也可能是手机 Android 版本太低（需要 Android 8.0 以上）。

**Q: 开启无障碍服务时系统警告"可能读取屏幕内容"？**
A: 这是正常提示，因为本应用确实需要截图分析。所有分析都在手机本地完成，不会上传到任何服务器，可以放心开启。
