# 短视频守护 (ShortVideoGuard)

一款 Android 无障碍辅助工具，在您浏览抖音、快手、TikTok 等短视频应用时，自动检测并遮挡擦边/不适宜内容，同时保留屏幕边缘滑动区域，让您可以继续切换视频。

## 功能特性

- 🤖 **自动检测**：通过无障碍服务实时监测屏幕内容，无需手动操作
- 🛡️ **智能遮挡**：当检测到不适宜内容时，自动在中央区域显示半透明遮挡层
- 👆 **保留交互**：屏幕左右两侧和底部保留透明区域，可正常滑动切换视频
- 🎚️ **可调敏感度**：在主应用界面可调节检测严格程度
- 🔄 **支持多平台**：兼容抖音、抖音极速版、TikTok、快手、快手极速版、小红书
- ⏸️ **临时解除**：遮挡层上提供"继续观看"按钮，可临时解除当前遮挡

## 工作原理

1. **无障碍服务**：注册为系统无障碍服务，监听目标短视频应用的界面变化
2. **实时截图**：使用 Android 原生截图 API（`takeScreenshot`）捕获当前屏幕
3. **内容分析**：在 HSV 颜色空间中检测肤色像素占比，结合空间分布特征计算不适宜内容置信度
4. **智能遮挡**：当置信度超过设定阈值时，通过悬浮窗覆盖半透明毛玻璃遮挡层
5. **边缘保留**：遮挡层避开屏幕边缘，确保用户可以继续滑动浏览

## 构建要求

- Android Studio Hedgehog (2023.1.1) 或更高版本
- JDK 17
- Android SDK API 34
- Kotlin 1.9.22

## 如何构建 APK

### 方法一：使用 Android Studio（推荐）

1. 下载本项目源码 ZIP 并解压
2. 打开 Android Studio，选择 **File → Open**，选择项目文件夹
3. 等待 Gradle 同步完成（首次可能需要下载依赖，约 5-10 分钟）
4. 连接手机或启动模拟器
5. 点击工具栏的绿色运行按钮（▶️），或选择 **Build → Build Bundle(s) / APK(s) → Build APK(s)**
6. 生成的 APK 位于 `app/build/outputs/apk/debug/app-debug.apk`

### 方法二：命令行构建

```bash
# 1. 进入项目目录
cd ShortVideoGuard

# 2. 使用 Gradle Wrapper 构建（首次会自动下载 Gradle）
./gradlew assembleDebug

# 3. APK 输出路径
ls app/build/outputs/apk/debug/app-debug.apk
```

### 方法三：生成签名 APK（用于发布）

```bash
# 1. 生成密钥库（只需执行一次）
keytool -genkey -v -keystore my-key.keystore -alias alias_name -keyalg RSA -validity 10000

# 2. 放置密钥库到项目根目录

# 3. 在 app/build.gradle.kts 中配置签名（或创建 signing.properties）

# 4. 构建发布版
./gradlew assembleRelease
```

## 安装与使用

### 首次使用设置

1. **安装 APK**：将 APK 文件传输到手机，点击安装（可能需要允许"未知来源"安装）

2. **开启无障碍服务**：
   - 打开「短视频守护」应用
   - 点击"前往开启无障碍服务"按钮
   - 在系统设置中找到「短视频守护」，将其开启
   - 确认弹出的权限警告

3. **授予悬浮窗权限**（部分系统需要）：
   - 前往系统设置 → 应用管理 → 短视频守护 → 权限
   - 开启"显示悬浮窗"权限

### 日常使用

1. 打开抖音、快手等支持的短视频应用
2. 正常浏览视频，当检测到不适宜内容时，屏幕中央会自动出现遮挡层
3. 如需切换视频，在屏幕**左右两侧**或**底部区域**正常滑动即可
4. 如果想看被遮挡的内容，点击遮挡层上的"继续观看"按钮（10秒后会恢复检测）

### 调节敏感度

- 打开「短视频守护」主应用
- 拖动"检测敏感度"滑块：
  - **宽松**：较少触发遮挡，可能漏掉一些边缘内容
  - **严格**：更容易触发遮挡，可能误挡正常内容

## 技术架构

```
ShortVideoGuard/
├── GuardAccessibilityService.kt    # 核心无障碍服务（事件监听+截图触发）
├── OverlayManager.kt               # 悬浮遮挡层管理（TYPE_APPLICATION_OVERLAY）
├── ContentAnalyzer.kt              # 图像内容分析（HSV肤色检测算法）
├── ScreenCaptureManager.kt         # 屏幕截图管理（无障碍API/MediaProjection备选）
├── MainActivity.kt                 # 主界面（Jetpack Compose）
└── ui/                             # UI组件和主题
```

### 检测算法说明

本项目使用**启发式肤色检测算法**（无需联网，纯本地计算）：

1. **颜色空间转换**：将 RGB 像素转换为 HSV 颜色空间
2. **肤色范围判断**：检测色调(H)是否在 0-50° 或 340-360° 范围，饱和度(S)和明度(V)是否在合理区间
3. **空间分布分析**：统计肤色像素在全屏的占比，以及在上半屏、中心区域的集中程度
4. **置信度计算**：综合肤色占比和空间分布，输出 0-1 的置信度分数

**优点**：无需下载大型 AI 模型，处理速度快（<100ms），离线可用，隐私安全
**局限**：相比深度学习模型精度略低，可能在以下场景误判：
- 沙滩/沙漠场景（大面积肤色相似色）
- 正常人物直播/自拍
- 暖色调滤镜视频

未来版本可接入 TensorFlow Lite 深度学习模型提升精度。

## 隐私说明

- **所有分析均在本地完成**，不会上传任何截图到服务器
- 无障碍服务仅在目标短视频应用前台时激活
- 截图数据在分析后立即释放，不做任何存储
- 应用不收集任何用户个人信息
